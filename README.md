# SchoolMonitor
利用AI制作的学校信息监控助手
===============================================================================
                     学校官网关键词监控助手
===============================================================================

一款自动抓取学校官网通知、按关键词筛选、生成摘要并通过桌面/微信推送的桌面工具。

人机协作开发实录：全程由用户提供需求，AI（DeepSeek）生成代码并迭代修复，3小时内
完成从想法到可交付exe的全过程。

-------------------------------------------------------------------------------
功能特性
-------------------------------------------------------------------------------

  - 多栏目监控：可同时监控多个列表页（如教务处、学院通知）
  - 关键词筛选：自定义关键词，只抓取你关心的内容
  - 自动摘要：基于 TextRank 算法生成中文摘要（2~3句话）
  - 去重处理：记录已处理文章，避免重复推送
  - 定时后台监控：自定义检查间隔（分钟级），独立线程不阻塞界面
  - 双通道通知：Windows 桌面弹窗 + 微信推送（集成 Server酱）
  - 图形界面：关键词管理、历史文章列表（支持过滤搜索）、双击打开原文链接、
              手动立即检查
  - 数据持久化：文章、关键词、已处理URL均存储在JSON文件中
  - 打包为单exe：无需Python环境，支持开机自启动


-------------------------------------------------------------------------------
快速开始
-------------------------------------------------------------------------------

环境要求（如果你是直接运行 .py 文件）：

  - Python 3.8+
  - 安装依赖（在命令行执行）：

    pip install requests beautifulsoup4 lxml sumy jieba plyer win10toast pillow
或者
    pip install -r requirements.txt

下载/克隆本项目：

    git clone https://github.com/yourname/school-monitor.git
    cd school-monitor

运行（源码方式）：

    python school_monitor_gui.py

打包为独立exe（可选）：

    pip install pyinstaller
    pyinstaller --onefile --windowed school_monitor_gui.py

生成的 school_monitor_gui.exe 位于 dist 文件夹，双击即可运行。

配置说明
-------------------------------------------------------------------------------

基本配置（外部配置文件，无需重新打包）

  程序启动时会自动读取同目录下的 config.json 文件。如果文件不存在，会自动
  创建一个默认模板。修改配置后重启软件即可生效。

  示例 config.json 内容：

    {
      "TARGET_URLS": [
        "https://jww.zjgsu.edu.cn/1331/list.htm",
        "https://jww.zjgsu.edu.cn/1335/list.htm"
      ],
      "KEYWORDS": ["奖学金", "竞赛", "实习", "考试", "转专业", "计算机", "放假"],
      "SERVERCHAN_SENDKEY": "你的SendKey",
      "CHECK_INTERVAL_MINUTES": 30,
      "MAX_AGE_DAYS": 30,
      "ENABLE_DESKTOP_NOTIFY": true
    }

  字段说明：

    TARGET_URLS           - 学校通知/新闻的列表页链接（支持多个）
    KEYWORDS              - 需要监控的关键词列表
    SERVERCHAN_SENDKEY    - Server酱的SendKey，留空则不推送微信
    CHECK_INTERVAL_MINUTES - 后台自动检查间隔（分钟）
    MAX_AGE_DAYS          - 只抓取最近N天内的文章（0表示不限制）
    ENABLE_DESKTOP_NOTIFY - 是否启用Windows桌面弹窗

高级定制：适配不同学校的网页结构

  如果程序抓取不到文章或摘要为空，需要根据目标学校的HTML结构调整代码中的
  两个函数：

    1. 链接提取规则：修改 extract_links_from_page 中的 if 条件，匹配该校
       文章链接的特征（例如是否包含 /info/、/20 等）。
    2. 正文选择器：修改 extract_article_content 中的 selectors 列表，填入
       文章正文的CSS选择器（可通过浏览器F12查看）。

文件说明
-------------------------------------------------------------------------------

  school_monitor_gui.py   - 主程序源代码
  config.json             - 外部配置文件（可选，优先于硬编码）
  keywords.json           - 关键词存储文件（GUI中修改时自动更新）
  articles.json           - 历史文章列表（标题、链接、摘要、时间）
  processed_links.json    - 已处理文章的URL哈希，防止重复通知
  关键词监控报告.html      - 手动点击“打开HTML报告”按钮生成

-------------------------------------------------------------------------------
常见问题
-------------------------------------------------------------------------------

1. 运行后提示 Chinese tokenizer requires jieba

   原因：sumy 处理中文需要 jieba 分词库。
   解决：执行 pip install jieba，然后重新打包（如果使用exe）。

2. 明明有最新文章，却提示“没有发现新文章”

   可能原因：
     - 链接提取规则不匹配 → 按“高级定制”调整 extract_links_from_page
     - 日期过滤太严格 → 在 config.json 中将 MAX_AGE_DAYS 设大或设为0测试
     - 关键词不匹配正文/标题 → 检查提取到的正文内容（可临时打印调试）

3. 桌面通知不弹出

    - 检查系统通知权限是否开启
    - 尝试安装 win10toast：pip install win10toast（代码已内置回退支持）
    - 确认 config.json 中 ENABLE_DESKTOP_NOTIFY 为 true

4. 微信收不到消息

    - 确认 SERVERCHAN_SENDKEY 填写正确
    - 确认已关注 Server酱 的公众号并绑定账号
    - 免费版每天有调用次数限制（通常够用）

5. 打包后exe运行报错缺少某个库

    - 确保打包环境中已安装所有依赖（特别是 jieba、sumy、plyer）
    - 建议使用虚拟环境（pipenv 或 venv）隔离打包环境

6. 如何设置开机自启动？

    - 最简单：将 exe 的快捷方式放入 shell:startup 文件夹
      （按 Win + R，输入 shell:startup）
    - 更可靠：使用 Windows 任务计划程序，设置触发器为“登录时”，
      并可添加延迟启动
